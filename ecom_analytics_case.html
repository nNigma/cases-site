Документация: CDI / BDI

CDI (Category Development Index) и BDI (Brand Development Index) — индексы “сильных” и “слабых” территорий/сегментов относительно среднего уровня. Их удобно использовать для поиска зон роста, приоритизации маркетинга/дистрибуции и планирования офлайн‑присутствия.

Интуиция (простыми словами)

  • CDI отвечает: “Насколько категория здесь развита относительно среднего?”
  • BDI отвечает: “Насколько бренд/товар здесь силён относительно среднего?”
  • Сравниваем не абсолютные продажи, а нормированный уровень (например, на человека или на аудиторию).

Формулы (базовый вариант)

CDI(region) = 100 * ( CategorySales(region) / Population(region) )
                    / ( CategorySales(total)  / Population(total) )

BDI(region) = 100 * ( BrandSales(region) / Population(region) )
                    / ( BrandSales(total)  / Population(total) )

На практике вместо Population можно использовать доступную “базу” — аудиторию, домохозяйства, активных покупателей, количество чеков и т.п. Важно, чтобы знаменатель был сопоставим между регионами.

Методология расчёта (шаги)

  • Выбрать “единицу нормализации”: население/аудитория/домохозяйства/покупатели.
  • Определить период: неделя/месяц/квартал, и единые фильтры.
  • Согласовать определение категории: правила категоризации, справочник, версии.
  • Посчитать продажи/объём по регионам и в целом.
  • Нормировать на базу (на 1 человека, 1 покупателя и т.д.).
  • Индексировать на общий средний уровень (×100).
  • Сегментировать регионы по квадрантам (см. ниже).

Как интерпретировать (квадранты)

  • CDI > 100, BDI > 100: сильная категория и сильный бренд → удержание/масштабирование.
  • CDI > 100, BDI < 100: категория развита, бренд слаб → зона атаки/доля рынка.
  • CDI < 100, BDI > 100: бренд сильнее категории → нишевое лидерство, аккуратно растить.
  • CDI < 100, BDI < 100: слабая зона → минимальная поддержка или “длинный рост”.

Практический совет: для устойчивости добавляйте пороги по объёму (например, минимум чеков/продаж), чтобы не “перекосило” индекс на малых числах.

Документация: карты для выбора офлайн‑точки

Цель карты — помочь выбрать район/локацию для открытия офлайн‑точки, учитывая маршруты, население и потенциальный спрос. Важно: карта — это инструмент приоритизации; финальное решение дополняется полевыми проверками (конкуренты, арендные ставки, трафик по времени суток, ограничения).

Что учитывается

  • Маршруты/перемещения (как люди реально перемещаются по городу)
  • Плотность населения и распределение по районам
  • Перекрытие текущей сети (где уже есть точки/партнёры)
  • Сценарии “зоны охвата” (радиусы, изохроны, “время‑до‑точки”)

Методология (упрощённо)

  • Разбиваем город на клетки/районы (геосетка или административные границы).
  • Считаем потенциал: население × коэффициенты спроса/активности.
  • Добавляем маршруты: где поток людей “проходит”, а не только “живёт”.
  • Снимаем перекрытие: не открываем точку там, где зона уже закрыта существующими точками.
  • Ранжируем районы по интегральному скору и объясняем вклад факторов.
